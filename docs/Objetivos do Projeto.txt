Objetivo Geral
Desenvolver uma solução baseada em ciência de dados para prever o risco de falha acadêmica de alunos em uma instituição de ensino, permitindo intervenções proativas por parte da instituição.

Criar um Jupyter Notebook para realizar uma análise exploratória dos dados (EDA - Exploratory Data Analysis).

- Identificar padrões, tendências e correlações entre variáveis.

- Detectar valores ausentes, outliers e possíveis inconsistências na base de dados.

Desenvolver um Jupyter Notebook para tratar os dados, incluindo:

- Remoção ou imputação de valores ausentes.

- Codificação de variáveis categóricas (e.g., one-hot encoding ou label encoding).

- Normalização e/ou padronização de dados numéricos.

- Seleção de características relevantes para a modelagem.

- Divisão da base em conjuntos de treino e teste.

Criar um Jupyter Notebook para comparar duas abordagens de modelagem:

- Regressão Logística: Como uma abordagem explicativa e linear.

- Random Forest: Como uma abordagem mais robusta e não linear.

- Realizar ajustes de hiperparâmetros para ambos os modelos.

- Salvar o modelo final treinado para uso posterior.

Construir um Jupyter Notebook para avaliar o desempenho dos modelos treinados, utilizando métricas como:

- Acurácia.
- Precision, Recall e F1-Score.
- Curva ROC e AUC.
- Matriz de Confusão.

- Comparar os modelos para identificar aquele com melhor desempenho e adequação ao problema.

Implementar uma aplicação web utilizando Streamlit, que deve:

- Permitir o upload de uma nova base de dados.

- Aplicar o modelo treinado aos novos dados.

- Exibir os resultados com a identificação dos alunos mais propensos a falhar.

- Fornecer visualizações ou relatórios resumidos que ajudem os responsáveis pela instituição a entender os padrões.
